1. Reconstrucción de cadena (greedy)

Python — build_chain_simple()
1. Deduplicar segmentos (quitar invertidos con mismos extremos)
2. Elegir segmento más cercano al mojón km=0 (punto de inicio)
3. Loop greedy: conectar el segmento cuyo extremo esté más cerca del último punto
4. Calcular acumulado Haversine acc[] en km reales

Relevamiento de Rutas y Caminos · App de Campo · v6.3

PWA móvil para relevamiento vial en campo — GPS, cámara, modo offline y sincronización con la nube
Funcionando en campo

Relevamiento de Rutas y Caminos es la app móvil compañera del Sistema de Relevamiento Vial DVBA Zona VI. Permite al personal técnico registrar el estado de rutas provinciales directamente desde el campo, con GPS, fotografía con sello institucional y funcionamiento completo sin conexión a internet. Es una Progressive Web App (PWA) servida desde el mismo servidor HTTP que la app de escritorio, accesible desde cualquier navegador vía HTTPS (ngrok).

Funciones principales

📡
GPS en tiempo real
watchPosition continuo desde que abre la app. Barra superior con coordenadas, precisión en metros y altitud. Botón ↓ Usar confirma y clava las coords en el formulario. Indicador LED en el header: naranja = buscando, verde = fijado.
📷
Cámara getUserMedia
Stream de video directo via getUserMedia. Captura del frame al canvas. Evita el uso de capture="environment" que crashea la RAM en Samsung. Soporta flip entre cámara frontal y trasera.
🔖
Sello institucional
Logo DVBA circular + Ruta/Partido/Progresiva + coordenadas GPS + fecha y hora en formato 24h + nombre de la app. Banda compacta en la parte inferior de la foto. Se aplica al guardar online y al sincronizar offline.
📴
Modo offline completo
Sin red: guarda registros y fotos en IndexedDB. Con red: sync automático via Background Sync del Service Worker o manual con botón ↑ Sync. El formulario se libera inmediatamente para el siguiente registro.
📋
Cola de pendientes
Modal con lista de registros pendientes: ruta, tipo, partido, km, estado, foto (thumb), fecha. Borrado individual. Botón Sincronizar todo con feedback de éxito o error visible en pantalla.
📲
PWA instalable
Instalable en Android via Chrome (menú → Agregar a pantalla de inicio). Íconos PNG externos en icon-192.png / icon-512.png — nunca en base64 en el manifest, evita crash en Samsung.
🔄
Actualizaciones controladas
Banner naranja persistente cuando hay nueva versión disponible. El usuario elige cuándo actualizar tocando el botón. El Service Worker no aplica skipWaiting automáticamente — evita recargas inesperadas en campo.
🗺️
Geocodificación
Con red, consulta Nominatim/OSM para mostrar el nombre de la calle o ruta detectada automáticamente por GPS. Solo informativo, no reemplaza la selección manual de ruta y partido.

Stack técnico

🏗️
HTML/CSS/JS puro
Sin frameworks. Una sola archivo dvba_campo.html autocontenido. Fuente Encode Sans, colores institucionales DVBA (#009aae).
⚙️
Service Worker
sw.js con cache de assets estáticos, Background Sync para cola offline, y listener manual de skipWaiting. El HTML nunca se cachea — siempre se descarga fresco.
🗄️
IndexedDB
Dos stores: cola (pendientes de sync) y hoy (historial local del día con thumbnails). Más robusto que localStorage para fotos en base64.
☁️
Supabase
Tabla relevamientos — misma que la app de escritorio. Bucket relevamientos/fotos/ para imágenes con sello. Campo progresiva es NOT NULL → se manda '' si está vacío.

GitHub Pages · Hosting sin servidor

Migración completa — las apps corren en la nube, sin PC encendida
Migrado

Ambas apps (dvba_campo.html y dvba_zona6.html) son archivos HTML estáticos autocontenidos. Migrando a GitHub Pages quedan disponibles 24/7 con HTTPS gratuito, sin depender de que la PC de la oficina esté encendida ni de ngrok.

💡
¿Por qué funciona? Las apps se comunican directamente con Supabase desde el navegador del usuario. No hay backend propio — el único servidor es Supabase, que ya está en la nube.
📱
PWA, GPS y modo offline: GitHub Pages sirve con HTTPS, que es el requisito para que Chrome habilite la API de Geolocation y el Service Worker. La app de campo instalada como PWA funciona exactamente igual que antes.

Flujo de trabajo para releases

Actualizar una versión — 3 pasos
# 1. Modificar el HTML y el SW con la nueva versión
#    dvba_campo.html: APP_VER='v8'
#    sw.js:          CACHE_NAME='dvba-campo-v8'

# 2. Subir a GitHub
git add dvba_campo.html sw.js
git commit -m "v8 — descripción del cambio"
git push

# 3. GitHub Pages publica en ~30 segundos
#    Los celulares de campo verán el banner naranja: v7 → v8
⚠️
Checklist por release: (1) APP_VER='vX' en dvba_campo.html · (2) CACHE_NAME='dvba-campo-vX' en sw.js · (3) git push. El banner de actualización v(X-1)→vX aparece solo en los dispositivos de campo.

servidor.bat · Arranque con un click

Windows Terminal con pestañas — cero ventanas flotantes
Implementado v6.5.2
servidor.bat — fragmento clave
:: Detectar Windows Terminal
where wt >nul 2>&1
if %errorlevel% == 0 goto :usar_wt

:: 3 pestañas en una sola ventana
wt ^
  new-tab --title "Servidor HTTP" ^
    cmd /k "cd /d C:\DVBA\app && python servidor_http.py" ^; ^
  new-tab --title "ngrok Tunel" ^
    cmd /k "timeout /t 2 && ngrok http --domain=incorporating-winterishly-ileana.ngrok-free.dev 8080" ^; ^
  new-tab --title "DVBA Info" ^
    cmd /k "echo URLs de acceso..."
✅
Windows Terminal viene preinstalado en Windows 11. En Windows 10: Microsoft Store → buscar "Windows Terminal" → instalar gratis.
